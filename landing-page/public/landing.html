Integration Guide

Add an AI chat widget to any webpage in 3 steps. Conversations stay private inside an iframe — your page only sees tool calls.

1. Add the Widget & Your Agent Key

Paste this into your HTML. Replace the key with the one from the API.

<!-- Load the Ozwell chat widget -->
<script src="https://your-server.com/embed/ozwell-loader.js"></script>

<!-- Configure it with your agent key -->
<script>
  window.OzwellChatConfig = {
    apiKey: 'agnt_key-your-agent-key-here'
  };
</script>

That's it for a basic chatbot! The agent key tells Ozwell which personality, model, and tools to use — all configured on the server when you create the agent.

2. Define Your Tools (if your agent uses them)

If your agent has tools (like reading form fields or updating the page), you need to tell the widget what those tools look like so the AI knows how to call them. Add a tools array to your config:

window.OzwellChatConfig = {
  apiKey: 'agnt_key-...',
  tools: [{
    type: 'function',
    function: {
      name: 'get_form_data',
      description: 'Read the current form values',
      parameters: {
        type: 'object',
        properties: {}
      }
    }
  }]
};

No tools? Skip this step entirely — Step 1 is all you need.

3. Handle Tool Calls on Your Page

When the AI decides to use a tool, the widget sends a message to your page. You run the tool and send the result back:

window.addEventListener('message', (e) => {
  // Only handle messages from the Ozwell widget
  if (e.data.source !== 'ozwell-chat-widget' ||
      e.data.type !== 'tool_call') return;

  let result = {};

  if (e.data.tool === 'get_form_data') {
    // Read values from your page
    result = {
      name: document.getElementById('name').value,
      email: document.getElementById('email').value
    };
  }

  // Send the result back to the widget
  OzwellChat.iframe.contentWindow.postMessage({
    source: 'ozwell-chat-parent',
    type: 'tool_result',
    result: result
  }, '*');
});

No tools? Skip this step too.
